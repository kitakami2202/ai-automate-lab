## 2. SEOHead.astro（全ページ共通）

AI Automate Lab と同一構成で自動生成：
```
<head>
  ├── <title> + <meta name="description">
  ├── <link rel="canonical">
  ├── OGP（og:title, og:description, og:image, og:type）
  ├── Twitter Card
  ├── JSON-LD（記事タイプに応じて自動切替）
  └── <link rel="alternate" hreflang="ja">
</head>
```

---

## 3. JSON-LD 自動生成

frontmatterの`schema.type`に応じて自動で構造化データを生成：

| schema.type | 出力されるJSON-LD | 用途 |
|-------------|------------------|------|
| `Article` | BlogPostingスキーマ | 企業分析記事・業界概要 |
| `FAQPage` | FAQPageスキーマ（frontmatterのfaqから生成） | FAQ含む全記事 |
| `ItemList` | ItemListスキーマ | テーマ別比較・ランキング記事 |
| `HowTo` | HowToスキーマ（ステップ自動抽出） | 有報の読み方ガイド |

**全記事共通で追加されるスキーマ:**
- `BreadcrumbList`（パンくずリスト）
- `WebSite`（サイト情報 + 検索ボックス）
- `Organization`（サイト情報）

| ファイル | 生成方法 | 内容 |
|----------|----------|------|

## 5. AEO具体施策

| 施策 | 具体的な実装 |
|------|-------------|
| **結論ファースト** | 各企業分析記事の冒頭に「この会社が賭けているもの TOP3」ボックス |
| **FAQ構造化** | frontmatterのfaqを3-5個必須化、FAQPage JSON-LD出力 |
| **表形式の比較** | 比較記事・企業分析のセグメント表は必ずMarkdownテーブル |
| **llms.txt** | サイトルートに配置、記事追加のたびに更新 |
| **引用しやすい文体** | 「〇〇は△△です。」の断定＋構造化。「有報によると」で出典明示 |
| **ソース明記** | 全数値に事業年度を記載。EDINETリンクで一次情報への導線 |
| **免責の明示** | 各記事にdisclaimer。信頼性と正確性の両立 |

### AI引用されやすい記事パターン

| パターン | 例 | frontmatter設定 | AIが引用する理由 |
|----------|-----|-----------------|------------------|
| 定義型 | 「有価証券報告書とは？」 | schema: Article | 明確な答えがある |
| 比較型 | 「五大商社のR&D費比較」 | schema: ItemList | 表形式で整理されている |
| 手順型 | 「有報の読み方ガイド」 | schema: HowTo | ステップが構造化されている |
| 一覧型 | 「R&D費ランキング」 | schema: ItemList | リスト形式で網羅的 |
| 分析型 | 「トヨタの有報分析」 | schema: Article + FAQ | 一次データに基づく独自分析 |

## 6. コンテンツ品質ルール

### 記事品質ルール（AI Automate Lab準拠＋有報×就活固有）

1. **結論ファースト**: 冒頭に「この会社が賭けているもの」を要約
2. **定義文を含む**: 各セクション冒頭に「〇〇とは、△△です。」
3. **H2は5つ以内**: 深くなりすぎない（H3で補足）
4. **表を2つ以上含む**: セグメント表・比較表はAIが構造認識しやすい
5. **FAQ 3-5個**: 記事末尾に必ず配置
6. **内部リンク3本以上**: 同カテゴリ2本 + 別カテゴリ1本
7. **Layer間導線**: Layer 2記事は冒頭にLayer 1（読み方ガイド）リンク
8. **数値に出典**: 全ての数値に「（20XX年3月期有報）」を明記
9. **「意味の翻訳」必須**: 数値の後に「だからキャリア選択にどう関係するのか」を添える
10. **有報の限界を明示**: わからないこと（社風等）は正直に書く
11. **免責事項**: 各記事にdisclaimer表記を含める
12. **コードブロック**: 不要（AI Automate Labとの差分）

## 7. コンテンツ更新ポリシー

### リライト判断基準（AI Automate Lab準拠）

| トリガー | 条件 | アクション |
|----------|------|------------|
| **有報更新** | 対象企業の新年度有報がEDINETに公開 | データリフレッシュ + 記事リライト |
| **順位下落** | 掲載順位が10位以上下落し2週間戻らない | 内容の加筆・最新情報への更新 |
| **CTR低下** | 表示回数はあるのにCTRが2%未満 | title / descriptionの改善 |
| **就活シーズン** | 毎年1-3月 | 面接活用セクションの強化 |
| **定期見直し** | 公開から6ヶ月経過 | 内容の正確性を確認 |

### 年版記事の扱い
- タイトルに年号を入れる記事は**ランキング系のみ**に限定
- URLには年号を**含めない**（SEO評価リセット回避）
- `updatedAt`のfrontmatter更新 + 有報の事業年度表記で対応
