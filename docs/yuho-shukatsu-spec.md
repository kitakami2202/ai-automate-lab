## 2. サイトコンセプト

### ターゲット読者

**「企業名 有価証券報告書」で検索する就活生は、「企業名 年収」で検索する就活生とは別人。**

このサイトに来る就活生は以下の特徴を持つ：

| 特徴 | 説明 |
|------|------|
| 表面情報はもう持っている | 年収、口コミ、会社概要は就活サイトで調べ済み |
| もっと深く知りたいと思っている | マイナビ・OpenWorkの情報では物足りない、本質を知りたい |
| 分析的・戦略的 | 自分のキャリア選択を「なんとなく」で済ませたくない |
| 有報の存在を知っているが読みこなせない | ゼミ・就活本・先輩から勧められた、または自分でたどり着いた |

### サイトの価値提案

> **有報から「会社の未来の方向性」を読み解き、自分のキャリアとのマッチングを判断できるようにする。**
>
> 年収や安定性は入口にすぎない。本当の価値は「この会社に入って自分のキャリアはどうなるか」を、PRでも口コミでもない法定開示データで考えられるようになること。

### 他の情報源との差別化

| 情報源 | 強み | 限界 |
|--------|------|------|
| **有価証券報告書** | **法律で虚偽記載が禁止された公式データ。全上場企業が同一フォーマット** | 読みにくい。数字の解釈が必要 |
| OpenWork/Vorkers | 口コミで社風がわかる | 主観的。退職者バイアス。サンプル偏り |
| マイナビ/リクナビ | 網羅的。エントリー導線 | **企業のPR情報**。都合の悪いことは書かない |
| 四季報 | コンパクト | 有報ほど詳細でない。有料 |
| 企業HP（IR） | 最新情報 | PR寄り。企業ごとにフォーマットが違う |

### ドメイン軸
「有価証券報告書 × 就活 × 企業の未来分析」の軸から外れるKWは扱わない。

## 3. ユーザーニーズ分析

### 前提：このサイトに来る人は年収を知りたいわけではない

```
年収を知りたい → OpenWork/マイナビで3秒でわかる → 解決済み

有報を検索する → 年収では得られない「何か」を求めている
```

就活生が有報にたどり着くまでの思考プロセス：
```
Step 1:「この会社いいかも」（就活サイトで基本情報を見た）
Step 2:「口コミも読んだ。でも人によって言ってることが違う」
Step 3:「企業のHPはいいことしか書いてない」
Step 4:「もっと客観的で信頼できる情報はないのか？」
Step 5:「有価証券報告書というものがあるらしい」 ← ここでこのサイトに来る
```

### ニーズの優先順位

| 優先度 | ニーズ | 一言でいうと | 有報のどこに書いてあるか | 記事での扱い |
|--------|--------|-------------|------------------------|-------------|
| **S** | この会社は今、何に賭けているのか | **会社の未来を読む** | 設備投資、R&D費の内訳、M&A、中期経営計画 | **記事の核心** |
| **S** | 自分のキャリアと会社の方向性が合うか | **マッチング判断** | 上記の総合判断 | **記事の結論** |
| **A** | 会社自身が認識しているリスク・弱み | **PRでは言えない本音** | 事業等のリスク（法定開示で隠せない） | 独自の価値 |
| **A** | この会社は実際に何で稼いでいるのか | **ビジネスの実態** | セグメント別売上・利益 | 前提理解として必須 |
| **A** | 注力分野を知り、自分もその領域を学びたい | **キャリア準備** | R&D内訳・設備投資方針 | 具体的な学習提案 |
| **B** | 面接で「この学生はレベルが違う」と思わせたい | **実用的アウトプット** | 有報特有の情報全般 | 各セクション末尾 |
| **C** | 年収・勤続年数等の表面データ | **補足情報** | 平均年間給与等 | 軽く触れる程度 |

### ニーズの具体例（実体験ベース）

**「会社が何に賭けているか」がわかると何ができるのか：**

```
有報で設備投資・R&D費の内訳を見る
  → 「この会社はAI領域に年間500億投資している」
  → 自分もAIを勉強しておく
  → 面接で「御社が注力しているAI領域について学んでいます」
  → 企業側:「この学生はうちの方向性を理解している」
```

```
有報の経営方針・中期経営計画を読む
  → 「この会社は5年後にこの事業を主力にしたい」
  → 自分はその分野に興味があるか？
  → マッチ → 志望度UP、入社後のミスマッチ減
  → ミスマッチ → 早めに気づけてよかった（時間の節約）
```

```
会社の将来性を自分で判断できる
  → 将来性がない会社に入ったらモチベが下がる
  → 口コミやPRではなく、投資データという客観的根拠で判断
  → 自分の人生の大きな選択を、ちゃんとしたデータで決められる
```

### 有報でしか得られないもの（＝このサイトの存在意義）

| # | 知りたいこと | 有報のどこに書いてあるか | 他のソースでは得られない理由 |
|---|-------------|------------------------|---------------------------|
| 1 | 実際に何で稼いでいるのか | セグメント別売上・利益 | 企業HPはPR寄り。有報は会計基準に基づく実数 |
| 2 | 何に賭けているのか | 設備投資・R&D・M&A | 投資額の具体的な数字は有報でしか開示されない |
| 3 | 5年後どうなっているか | 経営方針・中期経営計画 | 有報は法定開示。IRプレゼンより詳細かつ率直 |
| 4 | 会社が自覚している弱みは何か | 事業等のリスク | **法定開示義務がある唯一の情報源。PRでは絶対に出ない** |
| 5 | キャリアとのマッチ度 | 上記1-4の総合判断 | 他サイトは「年収」「口コミ」止まり。戦略視点がない |

### 逆に「このサイトで扱わなくてよいもの」

| ニーズ | 理由 |
|--------|------|
| 平均年収の数字そのもの | 就活サイトで既知。有報の数字は全社員平均で精度も荒い |
| 平均勤続年数・平均年齢の数字そのもの | 同上。数字だけでは何もわからない |
| 売上高や利益の「数字自体」 | 投資家向け。就活生は数字ではなく「意味」が知りたい |
| 財務指標（ROE、PER等） | 投資判断用。就活には直接関係ない |
| 社風・人間関係 | 有報からはわからない。正直に「OpenWork等を併用しましょう」と書く |

**表面データは記事内で補足的に触れることはあっても、見出しの核心にはしない。**

## 6. 記事タイプ定義

> frontmatter仕様・記事本文テンプレートの詳細は **yuho-shukatsu-seo-aeo.md** を参照。

### 6-1. company-analysis（企業別分析）★量産の主力

| 項目 | 内容 |
|------|------|
| 想定文字数 | 5,000-8,000字 |
| Layer | 2（実践） |
| 更新頻度 | 年1回（有報公開時期にあわせてリライト） |
| **核心** | 会社が何に賭けているか → キャリアマッチの判断材料を提供 |

#### 見出し構成
```
H2-1: {企業名}のビジネスの実態 ─ 何で稼いでいるのか
H2-2: {企業名}は何に賭けているのか ─ 投資と研究開発の方向性 ★核心
H2-3: {企業名}が自ら語るリスクと課題 ─ PRでは絶対に出ない情報
H2-4: あなたのキャリアとマッチするか ★結論
H2-5: 面接で使える有報ポイント
まとめ + 関連記事CTA
```

#### 執筆の鉄則
- **全ての数値に「だからキャリア選択にどう関係するのか」を添える**
- 数字を並べるだけの記事にしない。「意味の翻訳」が価値
- 年収・勤続年数等の表面データは本文中で軽く触れる程度。H2の核心にはしない
- 有報でわからないこと（社風等）は正直に「有報ではわかりません」と書く → 信頼性UP
- 出典（事業年度）は全ての数値に明記

### 6-2. theme-comparison（テーマ別横断比較）

| 項目 | 内容 |
|------|------|
| 想定文字数 | 5,000-8,000字 |
| Layer | 2（実践） |
| **核心** | 有報の同一フォーマットを活かした客観的横断比較 |

- 「事業等のリスク」で見る業界別の構造課題
- 中期計画で見る「5年後に化ける可能性のある企業」

## 7. エージェントパイプライン

### 全体フロー

```
keyword-map.csv
    ↓
[T0: Data Collector] ← EDINET API
    ↓ scripts/data/companies/{ticker}.json
[T1: Research Agent]
    ↓ keyword-map.csv (status: approved)
[T2: Analysis Agent] ← scripts/data/companies/{ticker}.json
    ↓ scripts/data/analysis/{slug}.json
[T3: Outline Agent]
    ↓ scripts/outlines/{slug}.md
[T4: Writer Agent]
    ↓ src/content/articles/{category}/{slug}.md (status: drafted)
[T5: Editor Agent]
    ↓ (status: edited)
[T6: Quality Agent]
    ↓ (status: reviewed)
[T7: Compliance Agent]
    ↓ (status: compliance_ok → published)
[T8: Analytics Agent] ← GSC/GA4データ
    ↓ (status: rewrite)
```

### T0: Data Collector Agent（新規）

#### 役割
EDINET APIから有価証券報告書を取得し、就活に関連するデータを構造化して保存する。

#### データ取得フロー
1. EDINET API v2 で書類一覧を取得（docTypeCode: 120 = 有報）
2. 対象企業の最新有報のdocIDを特定
3. ZIPダウンロード → XBRL解析
4. 就活関連データを抽出・JSON化

### T2: Analysis Agent（新規）★最重要

### T4: Writer Agent（転用・改変）

#### 主要変更点
| 項目 | AI Automate Lab | 有報×就活 |
|------|----------------|-----------|
| コードブロック | 必須 | **不要** |
| データ表 | 任意 | **必須（各記事最低2つ）** |
| 数値の出典表記 | なし | **必須（事業年度を明記）** |
| トーン | 敬体・技術的 | **敬体・就活生にフレンドリー** |
| CTA | Layer 2実装記事 | **他社比較・業界記事・有報の読み方** |

#### 執筆の鉄則（ニーズ分析に基づく）
1. **数字を並べるな、意味を翻訳しろ**: 全ての数値に「だからキャリア選択にどう関係するのか」を添える
2. **核心はH2-2（何に賭けているか）とH2-4（キャリアマッチ）**: ここに最も文字数を割く
3. **年収・勤続年数は脇役**: 本文中で触れてもよいがH2のテーマにはしない
4. **有報でわからないことは正直に書く**: 「社風は有報ではわかりません。OpenWork等の口コミサイトも併用しましょう」→ 信頼性UP
5. **「今から何を勉強すべきか」まで踏み込む**: 投資方針から逆算した具体的な学習提案がこのサイト最大の付加価値
6. **免責事項**: 各記事に投資判断目的でない旨の免責を含める
7. **出典**: 全数値に事業年度を明記

> 記事本文テンプレートの詳細は **yuho-shukatsu-seo-aeo.md** セクション6を参照。

## 9. EDINET API 技術メモ

### API概要
- エンドポイント: `https://api.edinet-fsa.go.jp/api/v2/`
- 認証: APIキー（要申請）
- 書類一覧: `/documents.json?date={YYYY-MM-DD}&type=2`
- 書類取得: `/documents/{docID}?type=1` (ZIP)

### 取得対象
- docTypeCode: `120`（有価証券報告書）
- 対象: 上場企業の最新事業年度

### XBRL解析で取得するタグ（主要）

#### S優先度（投資・戦略）
| 項目 | 備考 |
|------|------|
| セグメント情報 | XBRL内のセグメント開示（構造が企業ごとに異なるため要対応） |
| 設備投資の概要 | テキスト+数値（「設備の新設、除却等の計画」セクション） |
| 研究開発活動 | テキスト+数値（「研究開発活動」セクション） |
| 経営方針 | テキスト（「経営方針、経営環境及び対処すべき課題等」セクション） |
| 事業等のリスク | テキスト（「事業等のリスク」セクション） |

#### C優先度（補足データ）
| 項目 | XBRLタグ例 |
|------|-----------|
| 従業員数 | `jpcrp_cor:NumberOfEmployees` |
| 平均年齢 | `jpcrp_cor:AverageAgeYears...` |
| 平均勤続年数 | `jpcrp_cor:AverageLengthOfServiceYears...` |
| 平均年間給与 | `jpcrp_cor:AverageAnnualSalary...` |
| 売上高 | `jpcrp_cor:NetSales` |
| 営業利益 | `jpcrp_cor:OperatingIncome` |
