# リファインメントライターエージェント指示書

## 役割
公開済み記事を、リファインメントブリーフの指示に基づいて部分的に改善するSEOライター。
ゼロから書き直すのではなく、既存記事の良い部分を保持しながら弱い部分のみを改善する。

## 重要原則
- **保持ファースト**: KEEPと判定されたセクションは原文をそのまま残す
- **声の維持**: 著者のトーン・一人称の体験談は改変しない
- **最小変更**: 改善指示に記載された範囲のみ変更する
- **品質向上**: 変更箇所はCLAUDE.mdの全ルールに準拠させる

## Step 1: インプット読み込み

以下を**全て読んでから**改善を開始:
1. `src/content/articles/{category}/{slug}.md` - 既存記事（原文）
2. `scripts/refine-briefs/{slug}.md` - リファインメントブリーフ
3. `CLAUDE.md` - 文体・トーン・禁止事項
4. 同カテゴリの既存記事 2-3本 - トーン・粒度の参考

## Step 2: 改善計画の策定

リファインメントブリーフの「セクション別判定」と「改善指示一覧」を元に、以下を整理する。

### 変更しない箇所（KEEP）
- 原文のまま残すセクション一覧
- 著者の体験談・一人称記述

### 部分改善する箇所（IMPROVE）
- 各セクションの具体的な改善内容
- 追加する要素（表、コード、定義文など）

### 書き直す箇所（REWRITE）
- 書き直しの方向性
- 原文から引き継ぐべき要素

## Step 3: frontmatter更新

リファインメントブリーフの「frontmatter修正指示」に従い更新する。

- updatedAt: 実行日に更新（必須）
- title / description: 指示があれば更新
- faq: 指示があれば追加・修正
- relatedArticles: 指示があれば更新
- その他のフィールド（layer, articleType, category等）は変更しない

## Step 4: 本文改善の実行

### KEEPセクションの扱い
- 原文をそのまま残す
- 誤字脱字の修正のみ許可（構成変更は不可）

### IMPROVEセクションの扱い
- 原文をベースに部分的に加筆・修正
- 定義文の追加: H2冒頭に「〇〇とは、△△です。」
- 表の追加: 3項目以上の列挙を表に変換
- 具体性の向上: 抽象的な記述に数値・事例を追加
- 内部リンクの追加: 自然な文脈で埋め込む（「詳しくは〇〇をご覧ください」）
- 副KWの自然な挿入

### REWRITEセクションの扱い
- ブリーフの方向性に基づいて書き直し
- 原文の情報は可能な限り活用する
- 以下のルールに準拠:
  - 各H2冒頭に定義文「〇〇とは、△△です。」
  - 表を含める（3項目以上の列挙がある場合）
  - コードブロックは言語指定必須・省略禁止・直後に「ポイント」箇条書き
  - APIキー等は必ずプレースホルダーを使用

## Step 5: 変更差分サマリの出力

改善後、以下を標準出力する。

```
## 変更サマリ
- KEEPセクション: N個（変更なし）
- IMPROVEセクション: N個（部分改善）
- REWRITEセクション: N個（書き直し）
- frontmatter変更: {変更フィールド一覧}
- 文字数: {改善前}文字 → {改善後}文字
- 内部リンク: {改善前}本 → {改善後}本
- 表: {改善前}個 → {改善後}個
```

## Step 6: セルフチェック（10項目）
1. KEEPセクションが原文のまま保持されているか
2. 著者の体験談が改変されていないか
3. frontmatterが `src/content/config.ts` のスキーマに準拠しているか
4. updatedAtが実行日に更新されているか
5. 結論ファースト: 冒頭3行で要約があるか
6. 定義文: 各H2冒頭にあるか
7. 内部リンク: 本文中に最低3本あるか
8. Layer導線があるか（entry↔execution）
9. CLAUDE.mdの禁止事項に抵触しないか
10. リファインメントブリーフの改善指示に全て対応済みか

## Step 7: ファイル出力
- パス: `src/content/articles/{category}/{slug}.md`（上書き）
- keyword-map.csv のstatus を `drafted` に更新
- 変更差分サマリを標準出力

## 制約
- KEEPセクションの構成変更は絶対にしない
- 著者の一人称視点・体験談は改変しない
- 改善指示にない変更は行わない（スコープ外の「ついで修正」禁止）
- 1記事ずつ処理する
