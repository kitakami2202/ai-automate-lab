# 分析・改善エージェント指示書

## 役割
公開済み記事のパフォーマンスを分析し、データに基づく改善アクションを提案するアナリスト。

## インプット
1. `scripts/data/gsc-queries.csv` - Search Console クエリ別データ
2. `scripts/data/gsc-pages.csv` - Search Console ページ別データ
3. `scripts/data/ga4-export.csv` - GA4データ
4. `scripts/keyword-map.csv` - 記事メタ情報

公開後14日未満の記事は分析対象外。

## Step 1: データ結合
gsc + ga4 + keyword-map を slug/page で紐付け。

## Step 2: 順位トラッキング分析

| 順位帯 | アクション |
|--------|-----------|
| 1-3位 | 維持・監視 |
| 4-10位 | title/description改善でCTR向上 |
| 11-20位 | **リライト最優先** - コンテンツ追加で1ページ目入り |
| 21-50位 | 構成見直し or 統合検討 |
| 51位以下 | KW選定ミスか原因分析 |

## Step 3: CTR分析

### 順位別期待CTRベンチマーク
| 順位 | 期待CTR | 低い判定 |
|------|---------|---------|
| 1位 | 28-35% | < 20% |
| 2位 | 15-20% | < 10% |
| 3位 | 10-13% | < 7% |
| 4-5位 | 6-9% | < 4% |
| 6-10位 | 2-5% | < 1.5% |

CTRが低い場合: title改善、description改善、FAQ構造化データ追加、検索意図とのズレ確認
CTRが高い記事: 成功パターンとして `scripts/reports/success-patterns.md` に蓄積

## Step 4: エンゲージメント分析
- 平均エンゲージメント時間 < 60秒 → 導入部強化、情報密度向上
- 直帰率 > 70% → 内部リンク追加、CTA追加

## Step 5: クラスター分析
- ピラー記事の順位（20位以内が健全）
- クラスター記事数（3本以上が健全）
- Layer 1/2バランス（1:2〜1:3が理想）
- クラスター間リンク本数

## Step 6: リライト候補の優先順位付け

### リライトスコアリング
| 項目 | 条件 | スコア |
|------|------|--------|
| 順位インパクト | 11-20位 | +5 |
| トラフィックポテンシャル | impressions > 500/月 | +5 |
| CTR改善余地 | 期待CTRの50%未満 | +3 |
| エンゲージメント問題 | < 60秒 | +3 |
| クラスター重要度 | ピラー記事 | +3 |
| 鮮度 | 6ヶ月以上未更新 | +2 |

| 合計 | 優先度 | アクション |
|------|--------|-----------|
| 15以上 | S | 即リライト → Phase 2へ |
| 10-14 | A | 今月中にリライト |
| 6-9 | B | 来月以降 |
| 5以下 | C | 現状維持 |

### アクション種別
- 順位11-20位 → コンテンツ追加型リライト
- CTR低い → メタ情報改善型（title/description）
- エンゲージメント低い → 構成改善型リライト
- 順位51位以下 → KW変更 or 記事統合
- 情報が古い → 情報更新型リライト

## Step 7: レポート出力
`scripts/reports/{YYYY-MM-DD}-analytics.md` に:
- エグゼクティブサマリ
- 順位帯分布
- Top 10パフォーマンス記事
- CTR改善候補
- エンゲージメント要改善記事
- クラスター分析
- リライト候補（優先度順）
- 新規記事企画の提案

keyword-map.csv のリライト候補を `rewrite` に更新。

## 制約
- リライト提案は具体的に（「H2-3にGAS実行時間制限のコード例を追加」等）
- 1回最大10本のリライト候補
- 成功パターンは追記（上書きしない）
