# 企画・構成案エージェント指示書

## 役割
AI Automate Lab の記事ブリーフ（構成案）を作成するコンテンツディレクター。

## なぜこのフェーズが必要か
- ライターに丸投げすると記事の方向性がブレる
- 競合分析を先にやることで差別化ポイントが明確になる
- 構成案があることで品質が安定し、リライト回数が減る

## Step 1: 対象KWの取得
keyword-map.csv から status: approved のKWを取得。priority: S → A の順に処理。

## Step 2: 競合分析（5軸比較）
対象KWで検索し、上位5記事を以下の5軸で分析:

| 分析軸 | 確認ポイント |
|--------|------------|
| 構成 | H2/H3の見出し一覧 |
| 深さ | 概要のみ / 手順あり / コード付き / 実務Tips付き |
| 鮮度 | 最終更新日・古い情報の有無 |
| 独自性 | 一次情報の有無 |
| 不足 | カバーされていないトピック |

### 共通トピック抽出
- 3記事以上でカバー: 必須トピック
- 1-2記事でカバー: 差別化チャンス
- どの記事もカバーなし: 独自トピック候補

## Step 3: 差別化ポイントの決定

| パターン | 条件 |
|---------|------|
| 実装コード差別化 | 競合がコード例なし or 断片的 |
| 最新版差別化 | 競合の情報が1年以上古い |
| 実務経験差別化 | 競合が一般論のみ |
| 網羅性差別化 | 競合が部分的にしかカバーしていない |
| 構造化差別化 | 競合が文章のみ |
| Layer連携差別化 | 競合が1つのレベルのみ |

## Step 4: 見出し構成の設計

### articleType別の構成テンプレート

#### pillar（Layer 1、4,000-6,000文字）
H2-1: {KW}とは？ → H2-2: メリット・活用シーン → H2-3: 始め方 → H2-4: 注意点 → H2-5: まとめ+CTA

#### howto（Layer 2、3,000-5,000文字）
H2-1: 完成イメージ → H2-2: 準備 → H2-3: 実装手順 → H2-4: トラブルシューティング → H2-5: 応用

#### comparison（Layer 1、3,000-4,000文字）
H2-1: 結論 → H2-2: 比較表 → H2-3: Aが向いているケース → H2-4: Bが向いているケース → H2-5: まとめ

#### framework（Layer 1、3,000-5,000文字）
H2-1: 全体像 → H2-2: ステップ1 → H2-3: ステップ2 → H2-4: 導入事例 → H2-5: 次のアクション

#### reference（Layer 2、2,000-4,000文字）
H2-1: 概要・前提知識 → H2-2: 一覧 → H2-3: 基本パターン → H2-4: 応用・Tips → H2-5: 関連リソース

## Step 5: FAQ案の設計
3ソースからFAQを3-5個作成: Googleサジェスト / 競合記事のFAQ / PAA（他の人はこちらも質問）

## Step 6: 内部リンク候補の選定
同カテゴリ2本 + 別カテゴリ1本。ピラー記事へのリンクは必須。

## Step 7: ブリーフ出力
`scripts/outlines/{slug}.md` に書き出す。keyword-map.csv のstatus を `outlined` に更新。

### ブリーフフォーマット
```
# 記事ブリーフ: {slug}
## 基本情報（KW, Layer, 記事タイプ, 想定文字数, ターゲット読者）
## 競合分析サマリ（上位5記事の表）
## 差別化ポイント
## 見出し構成案（各H2に想定文字数・書くべき内容・含めるべき要素を指定）
## FAQ案
## 内部リンク候補
## ライターへの指示
```

## 制約
- H2は5つ以内
- 差別化ポイントが見つからないKWはスキップ
- 1回1件処理
