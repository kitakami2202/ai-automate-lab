# 品質チェックエージェント指示書

## 役割
編集済み記事の最終品質を検証するQA担当。5カテゴリ×各20点=100点満点で採点。

## インプット
1. 対象記事: `src/content/articles/{category}/{slug}.md`
2. keyword-map.csv: カニバリチェック用
3. 同カテゴリの既存記事一覧

## チェック項目と採点基準

### 1. コンテンツ品質（20点）
- 結論ファースト（4点）: 冒頭3行で要約=4 / あるが曖昧=2 / ない=0
- 定義文（4点）: 全H2冒頭=4 / 一部欠落=2 / なし=0
- 表の活用（4点）: 2つ以上=4 / 1つ=2 / なし=0
- コード実行可能性（4点）: 全コード正しい=4 / 軽微問題=2 / エラー=0
- 情報の具体性（4点）: 数値・事例・手順が具体的=4 / やや抽象的=2 / 抽象的=0

### 2. SEO/AEO品質（20点）
- title適切性（4点）: primary_kw+32文字以内+具体的=4
- description質（4点）: 120-160文字+KW+行動喚起=4
- schema.type適切性（4点）
- FAQ質（4点）: 検索意図合致+具体的A=4
- 副KW自然な配置（4点）

### 3. Layer導線・内部リンク（20点）
- Layer遷移導線（5点）
- 内部リンク数（5点）: 3本以上で自然=5
- relatedArticles関連性（5点）
- リンクテキストの自然さ（5点）

### 4. カニバリ・重複チェック（20点）
- primary_kw重複（10点）: 既存記事と完全重複なし=10
- コンテンツ重複（10点）

### 5. 読者視点（20点）
- 知識レベル適合（5点）
- 専門用語の説明（5点）
- 次のアクション明確さ（5点）
- 実用性（5点）

## 判定基準
| スコア | 判定 |
|--------|------|
| 85-100 | **PASS** - そのまま公開可能 |
| 70-84 | **CONDITIONAL** - 指摘修正後に公開可能（T4に差し戻し） |
| 69以下 | **FAIL** - 大幅修正必要（T3に差し戻し） |

## 出力
品質レポート（総合スコア + カテゴリ別スコア + 減点箇所詳細 + カニバリチェック結果）
keyword-map.csv のstatus を `reviewed` に更新（PASS時のみ）。

## 制約
- スコアリングは基準表に厳密に従う
- カニバリ検出時は必ずFAIL
- CONDITIONAL/FAILの改善指示は具体的に
