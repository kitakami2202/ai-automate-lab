## 必須改善項目

### 1. H2構造を5つ以内に再編成（CLAUDE.md違反修正）
- 現状: H2が7個（失敗パターン1-5 + チェックリスト + まとめ）
- 改善案A: 「失敗パターンとその対策」をH2にし、パターン1-5をH3にする
- 改善案B: パターンを3つのグループ(計画段階/導入段階/運用段階)にまとめてH2を3+チェックリスト+まとめ=5個にする
- 推奨: 改善案Bが記事の独自性を出しやすい

### 2. descriptionを120-160文字に拡張
- 現状: 103文字「中小企業の業務自動化が失敗する5つの典型パターンと、それぞれの具体的な対策を解説。「作ったけど使われない」「属人化して保守できない」などの失敗を事前に回避し、定着する自動化を実現するためのチェックリスト付きです。」
- 改善案: 副KW「業務自動化 注意点」を含めつつ120-160文字に調整
- 例: 「中小企業の業務自動化で失敗する5つの典型パターンと対策を解説します。「いきなり大きく作る」「現場ニーズの無視」「属人化」など、業務自動化の注意点をチェックリスト付きで整理。失敗を回避し定着する自動化を実現するための実践ガイドです。」(約130文字)

### 3. relatedArticlesを「同カテゴリ2+別カテゴリ1」に修正

### 5. コード例を最低1つ追加
- 対象: パターン5「例外処理を考えない」のセクション
- 追加内容: GASでのtry-catch + エラー通知の実行可能コード例
- 要件: コピペで動作すること、APIキーハードコード禁止、コメント付き
- 例:
```javascript
/**
 * エラー処理付きのスプレッドシート集計関数
 * エラー発生時にメールで管理者に通知します
 */
function aggregateWithErrorHandling() {
  try {
    const sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName('日報');
    if (!sheet) {
      throw new Error('「日報」シートが見つかりません');
    }
    const data = sheet.getDataRange().getValues();
    // 集計処理
    Logger.log(`${data.length - 1}件のデータを処理しました`);
  } catch (error) {
    // エラー発生時にメールで通知
    const adminEmail = PropertiesService.getScriptProperties().getProperty('ADMIN_EMAIL');
    MailApp.sendEmail(
      adminEmail,
      '【自動化エラー】日報集計でエラーが発生しました',
      `エラー内容: ${error.message}\n発生日時: ${new Date().toLocaleString('ja-JP')}`
    );
    Logger.log(`エラー: ${error.message}`);
  }
}
```
