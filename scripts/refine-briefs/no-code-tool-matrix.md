```yaml
# description を拡充（副KW「選び方」を含める）
description: "ノーコード自動化ツール（Zapier・Make・n8n・Power Automate）を用途×規模のマトリクスで比較。月間処理件数・連携サービス・社内スキルの3軸で最適なツールの選び方がわかるフレームワーク付き。無料プラン活用のコツも解説します。"

# FAQ追加（4-5個に増やす）
faq:
  - question: "ノーコードツールとGASはどう使い分けますか？"
    answer: "Googleサービス内で完結する処理（スプレッドシート集計・Gmail送信等）はGASが最適です。複数のWebサービスをまたぐ連携（フォーム→Slack→メール→CRM等）はノーコードツールが適しています。GASで基盤を作り、ノーコードで連携を広げる順番が効率的です。"
  - question: "無料プランだけでどこまでできますか？"
    answer: "Zapierは月100タスク・5ワークフロー、Makeは月1,000オペレーション・2シナリオ、n8nクラウド版は月5,000クレジットまで無料です。小規模な業務自動化の検証には十分ですが、本格運用では有料プランへの移行が必要になるケースが多いです（2026年2月時点）。"
  - question: "Power Automateも候補に入りますか？"
    answer: "Microsoft 365を利用中の企業にはPower Automateが有力な候補です。Microsoft製品との連携が手厚く、追加費用なしで使える場合もあります。ただし非Microsoft系サービスとの連携はZapierやMakeが優れています。"
  - question: "ノーコードツールの選び方で最も重要なポイントは？"
    answer: "月間処理件数・連携先サービス・社内スキルの3軸で判断します。月間件数が少なくシンプルな連携ならZapier、大量処理や複雑なフローならMakeかn8n、Microsoft環境ならPower Automateが最適です。まず無料プランで1ヶ月検証してから本格導入するのが堅実です。"
  - question: "複数のノーコードツールを併用するのはアリですか？"
    answer: "はい、用途に応じた併用は合理的な選択です。例えばシンプルな通知はZapier無料枠、複雑な集計はMake無料枠のように、各ツールの無料枠を組み合わせれば月額0円で複数の自動化を運用できます。"
```
