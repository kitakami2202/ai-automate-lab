プログラミングの知識がなくても、AIコーディングツールを使えば日本語の指示だけで業務自動化スクリプトを作成できます。
「コードを書く」のではなく「AIにやりたいことを日本語で伝える」だけです。
この記事では、事務担当者がゼロから最初の業務自動化を体験するまでの手順を、1つずつ丁寧に解説します。

### この記事の前提

| 項目 | 内容 |
|------|------|
| 対象読者 | プログラミング経験がない事務担当者・経営者 |
| 必要な知識 | パソコンの基本操作ができればOK |
| 所要時間 | 約1時間（環境構築20分 + 実践40分） |
| 費用 | Claude Pro $20/月 または API従量課金（1回数円〜） |
| 完成物 | CSVデータの自動集計 / ファイル名の一括変更 / メール下書き生成 |

## 非エンジニアでもAI自動化ができる時代

AIコーディングツールとは、人間が日本語（自然言語）で指示を出すと、AIがプログラムのコード（コンピュータへの命令文）を自動で作成・実行してくれるソフトウェアのことです。

### 従来との違い

これまでの業務自動化は「プログラミングができる人」に限られていました。AIコーディングツールの登場で、この前提が変わっています。

| 項目 | 従来の自動化 | AIコーディングツール |
|------|-----------|-------------------|
| 必要なスキル | Python・VBA等の文法知識 | 日本語で「やりたいこと」を説明できる |
| 作業のイメージ | コードを1行ずつ書く | 「〇〇して」とAIに伝える |
| エラーが出たら | エラー内容を自分で読み解く | AIに「エラーが出た」と伝えると自動修正 |
| 学習コスト | 数ヶ月〜数年 | 本記事の1時間で最初の体験ができる |

### この記事で使うツール

本記事では**Claude Code**を使います。Claude Codeは、ターミナル（黒い画面のアプリケーション）で動作するAIコーディングツールです。

| ツール | 特徴 | 選んだ理由 |
|--------|------|-----------|
| Claude Code | ターミナルで日本語指示 → コード生成・実行 | 日本語の理解力が高く、ファイル操作まで一括で行える |

他のツールとの比較は[AIコーディングツール比較](/articles/reviews/ai-coding-tools-comparison)をご覧ください。

## 準備 — 必要なのは3つだけ

準備とは、Claude Codeを使い始めるために必要なソフトウェアとアカウントを整える作業のことです。手順どおりに進めれば約20分で完了します。

### 1. Node.jsをインストールする

Node.js（ノードジェイエス）とは、Claude Codeを動かすために必要な土台となるソフトウェアです。

1. [Node.js公式サイト](https://nodejs.org/)にアクセスします
2. 緑色の「LTS」ボタンをクリックしてダウンロードします
3. ダウンロードしたファイルをダブルクリックし、画面の指示に従ってインストールします

インストールが完了したら、ターミナルを開いて確認します。

**ターミナルの開き方:**

| OS | 操作 |
|----|------|
| Windows | スタートメニューで「PowerShell」と検索して開く |
| Mac | Spotlight（⌘+スペース）で「ターミナル」と検索して開く |

黒い画面（ターミナル）が開いたら、以下を入力して Enter キーを押します。

```bash
node --version
```

`v22.x.x` のようにバージョンが表示されれば成功です。

### 2. Claude Codeをインストールする

ターミナルに以下を入力して Enter を押します。

```bash
npm install -g @anthropic-ai/claude-code
```

文字がたくさん表示されますが、エラー（赤い文字の `ERR!`）が出なければ成功です。

### 3. Anthropicアカウントでログインする

ターミナルに以下を入力して Enter を押します。

```bash
claude
```

自動でブラウザが開き、Anthropicのログイン画面が表示されます。アカウントをお持ちでない場合は[console.anthropic.com](https://console.anthropic.com)で無料作成できます。ログインが完了すると、ターミナルに入力待ちの状態（`>`マーク）が表示されます。

これで準備完了です。

### 準備チェックリスト

| 項目 | 確認方法 | 完了 |
|------|---------|------|
| Node.jsインストール | `node --version` でバージョン表示 | □ |
| Claude Codeインストール | `claude --version` でバージョン表示 | □ |
| ログイン完了 | `claude` でプロンプト（`>`）が表示 | □ |

## 実践 — 日本語で指示して自動化する

実践とは、Claude Codeに日本語で指示を出して、実際に業務自動化スクリプトを作成・実行する作業のことです。3つの業務を順番に自動化していきます。

### 実践1: CSVデータの自動集計

事務作業で最も多い「CSVファイルのデータを集計する」作業を自動化します。

**手順:** ターミナルで `claude` を起動し、以下のように日本語で指示します。

```
> まず練習用のサンプルCSVデータを作ってください。
> - ファイル名: sales.csv
> - カラム: 日付, 担当者名, 商品名, 金額
> - 担当者: 田中, 佐藤, 鈴木 の3名
> - 20行程度のデータ
```

Claude Codeが `sales.csv` を自動で作成してくれます。続けて集計を指示します。

```
> sales.csv を読み込んで、以下の集計をしてください。
> - 担当者ごとの売上合計
> - 商品ごとの売上合計
> - 結果を sales_summary.csv に保存
> - 結果を画面にも見やすく表示
```

AIがPythonスクリプトを自動生成し、実行結果を表示してくれます。やることは**日本語で指示を入力するだけ**です。

**ポイント:** 結果がイメージと違った場合は、続けて修正指示を出せます。

```
> 金額にカンマ区切りを付けて、円マークも表示してください。
```

### 実践2: ファイル名の一括変更

「フォルダ内の大量のファイル名を一定のルールで変更する」作業を自動化します。手作業では1つずつ右クリック→名前の変更を繰り返す作業です。

```
> 練習用に test_files というフォルダを作り、
> その中に sample_001.txt から sample_010.txt まで10個のテキストファイルを
> 作成してください。各ファイルにはダミーの内容を入れてください。
```

ファイルが作成されたら、名前の一括変更を指示します。

```
> test_files フォルダ内のファイル名を以下のルールで一括変更してください。
> - 変更前: sample_001.txt → 変更後: 2026-02_report_001.txt
> - つまり先頭に今月の日付（2026-02）を付けて、sampleをreportに変える
> - 変更前と変更後のファイル名の対応表を画面に表示してから実行すること
```

Claude Codeが変更前後の対応表を表示し、確認を求めてくれます。承認すると一括で名前が変更されます。10ファイルでも100ファイルでも、指示は同じです。

### 実践3: メール下書きの一括作成

「顧客リストをもとに、1件ずつメールの下書きを作成する」作業を自動化します。

```
> まず以下の形式で顧客リストCSVを作成してください。
> - ファイル名: customers.csv
> - カラム: 会社名, 担当者名, メールアドレス, 前回購入商品
> - 5社分のサンプルデータ
```

```
> customers.csv の各顧客に対して、新商品案内メールの下書きを作成してください。
> - メールの文面は以下の条件で:
>   - 件名: 【新商品のご案内】{担当者名}様
>   - 冒頭で前回購入商品のお礼を述べる
>   - 新商品「業務効率化パッケージ」を案内
>   - です・ます調、200文字程度
> - 全員分の下書きを mail_drafts.txt に保存
> - 画面にも一覧表示
```

AIが各顧客の情報を反映した個別のメール文面を自動生成してくれます。手作業では1通ずつ顧客名や商品名を入れ替えていた作業が、数秒で完了します。

AIへの効果的な指示の出し方について詳しくは[プロンプト設計入門](/articles/frameworks/prompt-engineering-business)で体系的に解説しています。

## 動作確認・つまずいたときの対処法

トラブルシューティングとは、操作中に問題が発生したときの対処方法のことです。非エンジニアがつまずきやすいポイントと解決策をまとめました。

### よくあるつまずきと対処法

| つまずき | 状況 | 対処法 |
|---------|------|--------|
| ターミナルの開き方がわからない | 黒い画面を見たことがない | Windows: スタートメニューで「PowerShell」を検索。Mac: ⌘+スペースで「ターミナル」を検索 |
| `npm: command not found` | Node.jsインストール後に表示されない | ターミナルを一度閉じて開き直す。それでも解決しない場合はPCを再起動 |
| `claude: command not found` | Claude Codeがインストールされていない | `npm install -g @anthropic-ai/claude-code` を再実行 |
| ログイン画面が開かない | ブラウザが自動起動しない | 表示されたURLをコピーしてブラウザに手動で貼り付ける |
| AIの応答が英語で返ってくる | 日本語で指示したのに英語 | 「日本語で回答してください」と追加で指示する |
| 生成されたスクリプトがエラーで動かない | AIのコードにバグがある | 「エラーが出ました。修正してください」とそのまま伝える。AIが自動修正する |
| ファイルが見つからない | 保存場所がわからない | 「今いるフォルダのパスを教えてください」とAIに聞く |

### 最も大切なこと：AIにそのまま聞く

エラーが出たり、操作がわからなくなったりしても、Claude Codeにそのまま日本語で質問できます。

```
> エラーが出ました。何が問題ですか？
```

```
> さっき作ったファイルはどこに保存されましたか？
```

```
> 今の作業を最初からやり直したいです。
```

プログラミングの知識がなくても、「困っていること」を日本語で伝えればAIが対処してくれます。これがAIコーディングツールの最大のメリットです。
